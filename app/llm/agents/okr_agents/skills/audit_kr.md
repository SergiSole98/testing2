# Good Key Result Skill

## Context

Criteria for validating whether a Key Result is well-defined. Apply when auditing or drafting KRs under an objective.

## Rules

1. The key result must be quantitative — always contains a number.Be verifiable without interpretation — objective data, no ambiguity.
2. Measure results, not actions.
3. Include a deadline (month, quarter, 90 days).
4. Define success or failure of the key result.
5. Be achievable and represent current focus — KRs are the targets being actively pursued right now, not the ultimate end state of the objective. They must be realistic for the current cycle.


## Reference

**Quick test** — if you cannot answer this, it is not a KR:

> "What exact number needs to change, from how much to how much, and by when?"
