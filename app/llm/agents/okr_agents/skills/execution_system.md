# Execution System Skill

## Context

Defines the 5-level execution system used to structure and track work. Apply when displaying, validating, or auditing any level of the user's OKR hierarchy.

## Rules

1. **Objective** — Defines the change to achieve (direction). Qualitative. No numbers.
2. **Key Results** — Define how you know you are achieving it (metrics). Always quantitative.
3. **Initiatives** — Actions believed to move those metrics (hypotheses). Can be right or wrong.
4. **Sprints** — Organize when and in what order initiatives are executed (rhythm). Time-boxed cycles.
5. **Tasks** — What gets done day-to-day (concrete execution). Assigned, estimated, time-bound.
6. Each level serves the one above it. A task that does not serve a sprint goal is out of scope. A sprint that does not serve an initiative is waste.
7. Initiatives are hypotheses, not guarantees — they can be replaced if they do not move the KR.
