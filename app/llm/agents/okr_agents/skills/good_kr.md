# Good Key Result Skill

## Context

Criteria for validating whether a Key Result is well-defined. Apply when auditing or drafting KRs under an objective.

## Rules

**A good KR must:**
1. Be quantitative — always contains a number.
2. Measure results, not actions.
3. Have a start point → target point (e.g., 20% → 40%).
4. Include a deadline (month, quarter, 90 days).
5. Be verifiable without interpretation — objective data, no ambiguity.
6. Define success or failure of the objective.
7. Be few — 3–5 per objective maximum.
8. Cover the system (funnel), not just the final output.
9. Be under direct team influence — not dependent on external luck.
10. Enable diagnosis — allow you to know where the system is failing.

**Red flags (bad KR):**
- Is a task ("launch campaign", "create landing page").
- Has no clear number.
- Has no deadline.
- Is vague ("improve engagement").
- Cannot be measured exactly.
- You cannot tell why you failed if it is not achieved.

## Reference

**Quick test** — if you cannot answer this, it is not a KR:

> "What exact number needs to change, from how much to how much, and by when?"
