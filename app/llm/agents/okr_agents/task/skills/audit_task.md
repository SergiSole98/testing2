# Audit Task Skill

## Context

Criteria for validating whether a Task is well-defined. Apply when auditing or drafting tasks under an Initiative or as BAU work.

## Rules

1. Only audit tasks that are not completed. Skip any task marked as done.
2. Is a single, concrete, executable action.
3. Has a binary outcome: done or not done.
4. Has exactly one owner.
5. Has a maximum duration of 2 hours — split if longer.
6. Has an Asana tag: INIT (linked to an initiative) or BAU (no initiative).
7. If tagged INIT → references the associated initiative explicitly.
8. Has a defined priority within the day.
9. Has no blocking dependencies at execution time.
